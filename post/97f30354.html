1. 术语和数据结构

术语

Graph由nodes(vertices)和edges构成，而路径指从一个节点到另一个节点所经过的边。如果图中所有节点间都存在路径，则图是连通的。
树是一种特殊的图，在树中不存在环(cycles)。
有向图中的边都是单向的，而有权图中的边是有权重的。

数据结构

Graph的表示方法有很多，通常会依据图的不同大小和处理方法来选择数据结构。以下是3种常用的表示方法：

Adjacency List

通过将所有的边存入一维列表中表示图。

// 无权图
vector<int> adj[N];
adj[1].push_back(2);
adj[2].push_back(3);
adj[2].push_back(4);

// 有权图
vector<pair<int, int> > adj[N];
adj[1].push_back({2, 5});
adj[2].push_back({3, 7});
adj[3].push_back({4, 6});

遍历方法：

for(auto u : adj[s]) {
    // process node u
}

Adjacency Matrix

使用矩阵存储图的边信息。

int adj[N][N];

Edge List

边列表包含图中所有的边信息。

// 无权图
vector<pair<int, int> > edges;

// 有权图
vector<tuple<int, int, int> > edges;

2. DFS 和 BFS

DFS

DFS通过递归进行搜索，假设图的数据结构为adjacency list，则实现如下：

vector<int> adj[N];
bool vis[N];

void dfs(int s) {
    if(vis[s]) return;
    vis[s] = true;
    // process node s
    for(auto u : adj[s]) {
        dfs(u);
    }
}

时间复杂度。

BFS

BFS以层序对图进行搜索，假设图的数据结构为adjacency list，则实现如下：

vector<int> adj[N];
bool vis[N];
int dis[N];

void bfs(int s) {
    queue<int> q;
    vis[s] = true;
    while(!q.empty()) {
        int t = q.front(); q.pop();
        // process node t
        for(auto u : adj[t]) {
            vis[u] = true;
            dis[u] = dis[t] + 1;
            q.push(u);
        }
    }
}

3. 最短路径

3种算法的时间复杂度如下：

Bellman-Ford

这是一种单源最短路径算法，要求Graph中不能有负圈，因此可以用来检测负圈。
算法最初将起点的距离设为0，其余点距离设为正无穷。算法通过不断寻找最短边来减少路径距离。
假设数据结构为edge list，其中包含有权边tuple(a, b, w)，则实现如下：

/* vector<tuple<int, int, int> > edges; */
int dis[N];

void bellman_ford(int x) {
    for(int i = 1; i <= n; i++)
        dis[i] = INF;

    dis[x] = 0;
    for(int i = 1; i <= n-1; i++) {
        for(auto e: edges) {
            int a, b, w;
            tie(a, b, w) = e;
            dis[b] = min(dis[b], dis[a] + w);
        }
    }
}

该算法可通过测试第n轮循环是否更新dis来判别是否存在负圈。同时有一种利用队列的实现来提高效率（SPFA Algorithm）。

Dijkstra

Dijkstra同样也是单源最短路径算法，区别在于该算法要求图中不存在负边。
假设图的数据结构为adjacency lists，其中adj[a]为pair(b, w)表示有权边。

/* vector<pair<int, int> > adj[N]; */
int dis[N];
bool vis[N];

void dijkstra(int x) {
    priority_queue<pair<int, int> > q;

    for(int i = 1; i <= n; i++) dis[i] = INF;
    dis[x] = 0;
    q.pus_back({0, x});
    while(!q.empty()) {
        int a = q.top().second(); q.pop();
        if(vis[a]) continue;
        vis[a] = true;
        for(auto u : adj[a]) {
            int b = u.first, w = u.second;
            if(dis[a] + w < dis[b]) {
                dis[b] = dis[a] + w;
                q.push({-dis[b], b})
            }
        }
    }
}

Floyd-Warshall

该算法是一种多源最短路径算法，可以得到图中所有节点之间的距离。
算法首先初始距离矩阵dis[][]，其中dis[a][b]表示从a到b的距离。

int dis[N][N];

/* init */
for(int i = 1; i <= n; i++) {
    for(int j = 1; j <= n; j++) {
        if(i == j) dis[i][j] = 0;
        else if(adj[i][j]) dis[i][j] = adj[i][j];
        else dis[i][j] = INF;
    }
}

void floyd_warshall() {
    for(int k = 1; k <= n; k++) {
        for(int i = 1; i <= n; i++) {
            for(int j = 1; j <= n; j++) {
                dis[i][j] = min(dis[i][j], dis[i][k] + dis[k][j]);
            }
        }
    }
}

4. DAG 和 拓扑排序

DAG

有一类重要的图叫做有向无环图。与这类图有关的问题通常我们可以先进行拓扑排序，之后应用动态规划解决。

拓扑排序

首先将所有节点的状态置为0。在搜索的过程中将节点状态置为1，搜索完成后将节点的状态置为2。
在将节点状态置为2的同时，把节点序号存入一维列表中，最终倒序输出一维列表得到拓扑排序。

需要注意的是，拓扑排序的结果不唯一。

DP

得到DAG的拓扑排序后，可通过动态规划解决如下问题：

  • 从a到b的最短/最长距离
  • 有多少不同的路径
  • 路径中边的最多/最少数量
  • 哪些节点在所有路径中都出现了（关键节点）

例如，考虑计算从点a到点b的所有路径的数量。令paths(x)表示从点a到点x的路径数量，同时paths(a) = 1。则总数量可通过DP得到:

5. Successor Graph 和 Floyd判圈

Successor Graph

Successor Graph也是一种DAG。在这类图中，每个节点之后一个后继节点。在图中有一个或几个分量，每个分量由一个环和通向环的路径构成。

Successor Graph也会被称为functional graph，因为每个结点的后继都是确定的，可通过函数succ(x)计算得到。

定义succ(x, k)表示节点x的k步后继节点。通过预处理，可以将succ(x, k)的复杂度从降低为。

Floyd判圈

Floyd判圈的时间复杂度，但空间复杂度仅为，通过同时使用两个指针实现。

a = succ(x);
b = succ(succ(x));
while(a != b) {
    a = succ(a);
    b = succ(succ(b));
}

由于指针a走了k步，指针b走了2k步，求差后可以推出环的长度能够除尽k。如下求圈（环）的入口：

a = x;
while(a != b) {
    a = succ(a);
    b = succ(b);
}

求圈（环）的长度：

b = succ(a);
length = 1;
while(a != b) {
    b = succ(b);
    length++;
}

6. 生成树

生成树包含图中所有节点和一些边，在树中的任意两点能够连通。生成树的权值是树中所有边的权值之和。
最小生成树是所有可能的生成树中权值最小的一颗。最小生成树不唯一。

常用的两种生成树算法的复杂度如下：

Kruskal

Kruskal’s Algorithm

Kruskal算法通过贪心地将不构成环路的最小边进行合并得到最小生成树。

算法首先对所有边值进行排序，复杂度为。之后建立最小生成树。

for(...) {
    if(!same(a, b)) unite(a, b);
}

算法的关键在于如何高效地判断相接的两个树收否构成回路。如果使用DFS/BFS方法判断，则每次合并所需的时间复杂度为。通过使用并查集能够将复杂度降为，这样合并部分的复杂度为。

并查集

并查集提供两种级的操作：unite和find。unite合并两个集合，find查询元素所属的集合。

for(int i = 1; i <= n; i++) link[i] = i;
for(int i = 1; i <= n; i++) size[i] = 1;

int find(int x) {
    while(x != link[x]) x = link[x];
    return x;
}

bool same(int a, int b) {
    return find(a) == find(b);
}

void unite(int a, int b) {
    a = find(a);
    b = find(b);
    if(size[a] < size[b]) swap(a, b);
    size[a] += size[b];
    link[b] = a;
}

通过在find中引入路径压缩，能够进一步降低复杂度。

int find(int x) {
    if(x == link[x]) return x;
    return link[x] = find(link[x]);
}

Prim

Prim算法首先在树中加入任意节点，之后总是通过选择权值最小的边，在树中加入节点。
利用priority_queue能够将时间复杂度降为。

在竞赛中多数选手选择Kruskal’s Algorithm。
