1. 树的基本概念和算法

树是一个连通的无环图，由和节点和条边构成。任意两点之间都有唯一的一条路径连接。只有一个邻接节点的节点被称为叶子节点。

树的遍历

DFS

树的遍历比图简单一些，因为树中不存在环。

void dfs(int s, int e) {
    // process node s
    for(auto u : adj[e]) {
        if(u != e) dfs(u, s);
    }
}

例如，从节点x处开始搜索：

dfs(s, 0);

DP

动态规划能够在搜索的过程中计算一些有用的信息。例如，如下代码计算了每个树s中所存在子树数量count[s]。

void dfs(int s, int e) {
    count[s] = 1;
    for(auto u : adj[s]) {
        if(u == e) continue;
        dfs(u, s);
        count[s] += count[u];
    }
}

二叉树的遍历顺序

在二叉树中，每个节点都有一个左子树和一个右子树（子树可能为空）。对于二叉树的遍历有如下三种方式：

  • pre-order
  • in-order
  • post-order

如果已知前序和中序，便可准确的重构出这颗树。如果已知中序和后序，也可重构出这颗树。但仅有前序和后序却不能。

树的直径

树的直径指树中两节点间最长的距离。以下有两种复杂的算法能够得到树的直径。

DP方法

这种方法首先随机选定一个节点作为树的根，之后通过子树解决该问题。

在树根中每条路径都有一个最高节点。 对于每个节点计算如下两种信息：

  • toLeaf(x)：从x到叶子的最长距离
  • maxLength(x)：最高节点为x的最长路径距离。

DP方法能够在复杂度内计算以上两种信息。
首先是toLeaf(x)，可以从x的子节点中选择一个最长的toLeaf(c)，之后加一。
maxLength(x)则可以通过从子节点中选择最长的两个toLeaf(a)和toLeaf(b)计算得到。

DFS方法

通过两次DFS搜索，可以得到树的直径。
首先随机选择一个节点a作为树根，找到距离最远的节点b。
之后将b作为树根，找到距离最远的节点c，则bc的距离为树的直径。

最长距离

对于每个节点x计算以x为起点的最长距离maxLength(x)。这其中最长的maxLength就是树的直径，这个问题可以在复杂度内解决。

首先对于每个节点，计算子节点（downwards)方向上的最长距离maxLength，这可以通过DP在复杂度内解决。

之后计算父节点（upwards)方向上的最长距离maxLength，同样也是通过DP，但这里需要两个值：

  • maxLength_1(x)：从x到叶子的最长距离。
  • maxLength_2(x)：从x到叶子的最长距离，但与maxLength_1的方向不同。

这样，如果maxLength(x)不等于maxLength_1，则父节点方向上的最长距离为maxLength_1(x) + 1，否则为maxLength_2 + 1。

2. 树上查询

树上查询是针对有根树的查询，通常有关子树和树上路径，这些问题可以在常数或对数时间内解决。

寻找祖先

令ancestor(x, k)表示节点x的第k步祖先。朴素的方法是移动k步，复杂度为。通过预处理可以高效完成该问题。

计算k为2的幂的ancestor(x, k)，可以在总时间的复杂度内完成预处理，之后每次查询只需要的复杂度。

子树和路径

树遍历数组也就是用DFS遍历树节点的顺序，也就是DFS序。该数组的一个重要性质就是每颗子树在数组中都是一段以树根为起点的区间。

子树查询

假设树的每个节点都有一个值。并需要完成两种操作：更新一个节点的值；计算一个子树的和。

为了解决这个问题，需要对每个节点记录三个信息：

  1. 节点id
  2. 以该节点为根的子树的大小size
  3. 该节点的值value

通过这些信息，将子树所表示区间内的值相加可以得到子树的求和。

如果将以DFS顺序表示的value数组存入线段树或树状数组中，则可以实现在的复杂度内更新节点值和查询子树和。

路径查询

通过DFS序，可以高效的计算从根节点到任意节点的路径上的节点值之和。

例如，问题要求提供如下两种操作：更新一个节点的值（这将会影响该节点的所有子节点的路径和）；计算从根节点到任意节点的和。

为了解决这个问题，需要对每个节点记录三个信息：

  1. 节点id
  2. 以该节点为根的子树的大小size
  3. 从该节点到根的路径和。

该问题可以通过树状数组或线段树在复杂度内更新和查询，或通过差分数组在中更新，内查询。
